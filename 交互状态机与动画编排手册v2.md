## 二、状态机 v2.0 总览

### 2.1 核心理念转变

```
旧理解（v1.0 - 错误）：
  乌萨奇 = 温柔治愈的小可爱
  动作 = 微微呼吸眨眼、轻轻跳跃、乖巧吃东西
  
新理解（v2.0 - 用户校正）：
  乌萨奇 = 充满混乱能量的捣蛋鬼精灵
  动作 = 疯狂奔跑 --> 滑倒撞墙 --> 贴墙上挣扎 --> 扭屁股逗你 -->
         大喊"哈！！" --> 一屁股坐在地上 --> 爬起来继续跑
```

### 2.2 六大主状态

```
                    +==========================================+
                    |     乌萨奇状态机 v2.0（混乱能量版）        |
                    +==========================================+

   WALK_RUN          WALL_STUCK        ACTION_HA!
  （疯跑巡逻）      （撞墙贴墙）       （哈！爆发）
       |                 |                 |
       |    +------------+---------+      |
       |    |            |         |      |

   ACTION_EAT      SLIP_FALL    PEEK_INPUT  LIE_RARE
  （吃烦恼）       （滑倒爬行）   （偷看输入）（超稀少躺平）
```

### 2.3 状态转换矩阵

| 触发事件 | 当前状态 --> 目标状态 | 条件 | 优先级 |
|---|---|---|---:|
| 页面加载完成 | 无 --> WALK_RUN | 初始化 | -- |
| 用户点击乌萨奇 | ***任何*** --> ACTION_HA! | click/touchend | **P0 最高** |

| 任意交互唤醒 | LIE_RARE --> ACTION_HA! | 点击/触摸乌萨奇 | P0 |

## 三、各状态详细动画规格 v2.0

### 3.1 WALK_RUN -- 疯跑巡逻（默认常态）

**设计理念：** 乌萨奇不是在"走"，它是在**满屏幕疯跑**！偶尔会失控滑倒，撞到边界时会有夸张的撞墙反弹。这才是 Chiikawa 式的混乱能量感。

**视觉表现：**
- 以较高速度（2.0 ~ 3.0 px/frame）在屏幕内移动
- 主要使用奔跑帧 **[2][3]** 交叉切换（每 120 ~ 150ms 一帧）
- 移动方向偏向水平（正负60度以内），每 4 ~ 8 秒随机变向一次
- **偶尔（约 2~3%/秒的概率）触发失控滑倒** --> 进入 SLIP_FALL 状态
- **10% 的概率插入 [1][5] 停顿观察帧**（跑累了停下来看一眼，300ms 后继续跑）

**精灵图编排：**

| 子阶段 | 使用帧 | 播放节奏 | 叠加 CSS/JS 效果 |
|---|---|---|---|
| 向右冲刺 | `[2] --> [3] --> [2]...` 循环 | 每 130ms 切换 | scaleX(1) + run-bounce 抖动 |
| 向左冲刺 | `[2] --> [3] --> [2]...` 循环 | 每 130ms 切换 | scaleX(-1)镜像翻转 + run-bounce |
| 跑累停顿观察 | `[1]` 或 `[5]` | 保持 400ms | 速度归零，微微上下浮动 |
| 即将变向（预兆） | `[4]` 插入 1 帧 | 150ms | 身体前倾暗示要转向了 |

**CSS 运动叠加：**

```css
/* 奔跑时的腾空弹跳感 */
.usagi-running .usagi-sprite-wrapper {
  animation: usagi-run-bounce 0.2s ease-in-out infinite;
}
@keyframes usagi-run-bounce {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-6px); }  /* 跑步腾空 */
}

/* 方向翻转 */
.usagi-facing-left #usagi-sprite { transform: scaleX(-1); }
.usagi-facing-right #usagi-sprite { transform: scaleX(1); }
```

**音效：**
- 页面首次加载起步时：`start.wav`（音量 0.3）
- 其余时间静默奔跑（不需要每步出声，太吵）
- 变向时无声（自然过渡）

**行为规则：**
- z-index: 9（在内容层之上，UI 层之下）
- 夸夸气泡容器实时跟随当前位置
- 窗口 resize 时自动钳位到可视范围内
- **不遮挡 UI 元素**（如果跑到输入框附近，z-index 不超过 UI 层；或者通过碰撞检测绕开 UI 区域）

### 3.2 WALL_STUCK -- 撞墙贴墙

**触发时机：** WALK_RUN 状态下乌萨奇位置超出屏幕边界

**设计理念：** 不是简单的物理反弹，而是一套完整的"撞墙表演"——先夸张撞墙翻白眼，然后黏在墙上扑腾两下，再挣脱下来继续跑。这才是 Shimeji 的灵魂。

**完整时序编排：**

```
时间轴（总计 ~2000ms）：
=====================@============================>

  0ms       300ms       800ms       1400ms      2000ms
  |         |           |           |           |
  v         v           v           v           v
[撞击瞬间]  [翻白眼]   [粘墙上]    [挣扎]     [挣脱落下]
  |         |           |           |           |
  根据方向   [9]或[10]  [12]-[13]  [12]-[14]  [4]+[1]
  选帧：     (撞墙      (粘墙      (继续      (滑下来
  左墙用9    翻白眼)     闭嘴<->    扑腾)      恢复跑)
  右墙用10)              张嘴)
```

**根据撞墙方向的帧选择：**

| 撞向 | 撞击帧（0~400ms） | 贴墙帧（400~1400ms） | 挣脱帧（1400~2000ms） |
|---|---|---|---|
| **左边缘** | `[9]`（撞左墙翻白眼） | `[12]<->[13]<->[14]` 循环（粘左墙，闭嘴/张嘴交替） | `[4]`（滑落）→ `[1]`（恢复站立） |
| **右边缘** | `[10]`（撞右墙翻白眼） | `[12]<->[13]<->[14]` 循环（同样粘墙上，通过scaleX翻转呈现右侧感） | `[4]`→ `[1]` |
| **上边缘** | `[23]`（贴天花板！特殊处理） | `[23]<->[24]<->[25]`（贴天花板循环，扁扁的） | `[1]`（恢复正常） |
| **下边缘** | `[19]`或`[22]`（轻微滑倒） | `[20]<->[21]`（在地上爬/拱起<->趴平） | `[11]`（坐地上休息一下）→ `[1]` |

**音效：**

| 时机 | 音效 | 音量 | 说明 |
|---|---|---:|---|
| 撞击瞬间（0ms） | `sit.wav` | 0.6 | "咚！"撞墙声 |
| 贴墙扑腾中（800ms） | 无 | -- | 静默扑腾更有喜剧效果 |
| 挣脱落下（1500ms） | `start.wav` | 0.4 | "呀~"滑落的短促音 |
| 如果是撞下边缘（地板） | `sit.wav` | 0.5 | 摔在地上的声音 |

**GSAP 运动：**

| 时段 | 运动效果 | 参数 |
|---|---|---|
| 撞击瞬间 | 向反方向弹回一小段 | `yoyo: true, repeat: 1`, 距离 15px |
| 贴墙期间 | 微微颤动（像黏住了一样） | `x: ±2px` @ 150ms 循环, 3 次 |
| 挣脱时 | 向屏幕中心方向弹出 | `ease: "back.out(1.5)"`, 给一个弹射感 |

### 3.3 ACTION_HA! -- "哈！"爆发反应（点击核心交互）

**触发时机：** 用户单击/触摸乌萨奇本体

**设计理念：** 这是乌萨奇的**招牌动作**！点击它 = 它对你大喊一声"哈！！！"——充满能量、有点吓人但又很好玩。配合连击系统递进增强。

**完整时序编排（单次点击）：**

```
时间轴（总计 ~1500ms）：
=====================@======================>

  0ms      200ms      500ms      900ms      1300ms    1500ms
  |        |          |          |          |         |
  v        v          v          v          v         v
[蓄力]   [蓄力中]    [爆发!!!]  [保持]     [收尾]   [恢复]
  |        |          |          |          |         |
 [26]     [26]       [27]★      [27]       [28]     [idle]
 (抱胸闭嘴)(停顿)   (抱胸张嘴)  (张嘴延    (抱胸闭嘴) (回到跑)
                     "哈！！！"  长显示)
```

**关键细节 —— Frame 27 的时长控制：**
> 用户明确要求：**27 号帧的张嘴持续时间必须和音频中的"哈"声音等长！**
> 这意味着需要用代码精确控制：播放 `start.wav` 或 `double.wav`（ whichever 包含"哈"声），在 audio.duration 期间持续显示 frame 27，audio ended 后切到 frame 28。

**精灵图逐帧编排：**

| 时间段(ms) | 使用帧 | GSAP 身体运动 | 音效 |
|---|---|---|---|
| 0 ~ 300 | `[26]` 抱胸预备 | 微微前倾 `rotate: -3deg` → 回正 | 无（蓄力静默） |
| 300 ~ 300+T | `[27]` 张嘴爆发！！！ | **整体放大** `scale: 1 → 1.2`（弹性膨胀） + 轻微震动 `x: ±4px` | **播放"哈"音效**（start.wav 或 double.wav），音量 0.8 |
| 300+T ~ 300+T+300 | `[28]` 收尾 | scale 回到 1.0，加一个余震 `rotate: ±2deg` 衰减 | sit.wav（落地收势），音量 0.3 |
| 300+T+300+ | 回到 WALK_RUN 的 `[2][3]` | 恢复正常走动 | -- |

> 其中 T = "哈"音频的持续时长（毫秒）。建议预先测量：`new Audio('/assets/sounds/start.wav').duration`

**可选变体 —— "哈"完坐下：**

| 条件 | 动作 |
|---|---|
| 30% 概率（随机） | 28 之后接 `[30]`（一屁股坐在地上）保持 800ms，然后再站起来继续跑 |
| 70% 概率（默认） | 28 之后直接回到 WALK_RUN |

**连击系统 —— 快速连续点击：**

```
点击次数 --> 反应强度：

  第 1 次点击：
  --> 标准 "哈！" 反应（如上表，~1500ms）
      音效: start.wav

  第 2 次点击（距上次 < 1200ms 内）：
  --> 强化 "哈！！" 反应
      - [27] 显示时间延长 1.5x
      - scale 放大到 1.3（更夸张）
      - 屏幕加入轻微 shake（CSS 震动）
      - 音效: double.wav（更高亢）

  第 3 次点击（距上次 < 1200ms 内）：
  --> 超级 "哈！！！" + 扭屁股嘲讽
      - 先播放标准 HA!（26-27-28）
      - 紧接着无缝衔接 **扭屁股循环 [15]->[16]->[17]->[15]...** 持续 1.5s
      - 屏幕强烈 shake（幅度 ±5px × 4次）
      - 周围爆出 8~10 个黄色星星粒子（CSS 生成）
      - 音效: double.wav + start.wav 连续

  第 4+ 次点击（< 1200ms 内）：
  --> "受不了了！"
      - 直接切到 `[30]`（生气蹲/坐下）持续 2s
      - 气泡文字（可选）："别戳啦！(>_<)"
      - 进入 3s 冷却期不响应点击
      - 冷却结束后恢复正常响应
```

### 3.5 SLIP_FALL -- 失控滑倒与爬行

**触发时机：** WALK_RUN 中以约 2~3%/秒的随机概率触发（不是每次都滑倒！偶尔才有惊喜）

### 3.6 PEEK_INPUT -- 偷看输入框（不遮挡版）

**触发时机：** 用户 focus 了输入框（开始打字），且当前乌萨奇距离输入框 > 200px

**设计理念：** 乌萨奇注意到你在写字了，好奇地凑过来偷看——但它**不会遮挡你的输入框**。它会停在输入框旁边（左侧或右侧，选不遮挡的一侧），探着头看你写了什么。如果你写了很久，它可能会从站着变成坐下等着，甚至开始扭屁股打发时间。

**位置计算（防遮挡算法）：**

```
1. 获取 input 元素的 getBoundingClientRect()
2. 计算 input 的屏幕位置：inputLeft, inputTop, inputRight, inputBottom
3. 计算乌萨奇当前位置：usagiX, usagiY
4. 决定停靠侧：
   - 如果 usagiX < inputLeft --> 停在输入框**左侧**（目标x = inputLeft - 160）
   - 如果 usagiX >= inputRight --> 停在输入框**右侧**（目标x = inputRight + 20）
5. 目标 Y：与输入框上沿对齐（inputTop - 40），确保不遮盖文字
6. z-index 保持不变（低于 UI 层），确保输入框永远可点可编辑
```

**精灵图编排：**

```
时间轴（偷看全程，直到用户 blur 或提交）：
========================================@

  阶段1: 靠近（从当前位置走向输入框旁）
  ------
  使用帧: [2][3] 奔跑（速度 2.5x，表现出"好奇跑过来"的感觉）
  到达距离目标 30px 时减速

  阶段2: 探头偷看（初始 0~5 秒）
  ------
  使用帧:
    [5] 低头往下看输入框（主要帧，表示在看）
    偶尔插 [1] 站直发呆一下（300ms）
    偶尔插 [6] 换个角度看（往右下方瞥一眼）

  阶段3: 等得不耐烦了（5~15 秒后，如果用户还在打字）
  ----------
  使用帧过渡:
    [5] -> [7] 或 [8]（悬空坐下来等）
    然后 [7]<->[8] 来回摇晃（表示无聊等待）
    偶尔插 [15][16][17] 扭屁股三连（打发时间！很偶尔才做，约 8%概率/秒）

  阶段4: 兴奋 anticipation（检测到用户即将提交）
  --------
  当 input 字数 > 5 且最近 2 秒内有输入时:
    切换到 [26]（HA!预备）-- 身体紧绷准备中
    表示"有好戏看了！"
```

**音效：**

| 时机 | 音效 | 说明 |
|---|---|---|
| 开始跑向输入框 | `start.wav` vol 0.3 | 小步快跑的声音 |
| 到达停下 | `sit.wav` vol 0.2 | 轻轻停下的声音 |
| 等待期间扭屁股 | 无 | 静默更有喜感 |
| 检测到即将提交（进入 [26]） | 无 | 静默蓄力 |

**退出条件：**
- 用户 blur 输入框 --> 回到 WALK_RUN（跑开）
- 用户点击"粉碎它！" --> 进入 ACTION_EAT（吃烦恼流程覆盖此状态）
- 用户直接点击乌萨奇 --> 进入 ACTION_HA!（被戳了！吃完再做 HA!）

### 3.7 LIE_RARE -- 极稀有的躺平

**触发时机：** 连续 **90 ~ 180 秒**（随机）无任何用户交互

**设计理念：** 这是非常罕见的事件！用户要求"不要一会儿就睡觉了"。所以躺平应该是**惊喜彩蛋**——你可能开了这个页面一个小时才会偶然看到一次乌萨奇自己躺下了。而且躺的方式也是随机的搞笑风格。

**三种躺法随机选一：**

#### 躺法 A -- "贴天花板"（最搞怪）

```
使用帧: [23] -> [24] -> [25] 循环
表现: 乌萨奇不知怎么跑到屏幕顶部去了，
       整只兔子扁扁地贴在天花板上（或顶部边缘），
       一会闭嘴一会张嘴像在说话。
持续: 15 ~ 30 秒（随机），然后自动"掉下来"([4] -> [1])恢复跑动
音效: 贴上去时 sit.wav vol 0.3; 掉下来时 start.wav vol 0.4
```

#### 躺法 B -- "一屁股坐下生气"

```
使用帧: [26] -> [27](长) -> [28] -> [30]
表现: 乌萨奇跑着跑着突然停下来，
       对着空气喊了一声"哈！！"，
       然后一屁股坐在地上双手交叉生闷气。
       [30] 坐着的状态保持较久。
持续: [30] 保持 10 ~ 20 秒
音效: [27]时播放"哈"音效; [30]时安静
唤醒: 用户点击它 --> 立即进入 ACTION_HA!（又被戳到了！更生气地HA!）
```

#### 躺法 C -- "滑倒摆烂"

```
使用帧: [4] -> [18] -> [11] 长时间停留
表现: 跑着跑着突然 [4] 飞出去 -->
       [18] 直接滑倒了不动了 -->
       过渡到 [11] 盘腿坐地上发呆。
       整个角色静止不动，只有极微弱的呼吸起伏。
持续: [11] 发呆 15 ~ 25 秒
音效: [4]时无声; [18]时 sit.wav; 之后完全安静
```

**选择逻辑：** 每次 LIE_RARE 触发时，均等概率 1/3 选一种。**同一次会话内不重复同一种躺法。**

**全局冷却：** LIE_RARE 触发后，至少 300 秒内不再触发（防止频繁躺平打扰体验）。
