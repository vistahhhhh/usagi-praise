## 二、状态机总览（State Machine Architecture）

### 2.1 核心状态定义

乌萨奇在任何时刻都处于以下 **6 大主状态** 之一：

```
                    ┌─────────────────────────────────────────┐
                    │           乌萨奇状态机 v1                │
                    └─────────────────────────────────────────┘

     IDLE_WALK         IDLE_STAND       ACTION_EAT
    （巡逻漫游）      （静止站立）      （吃掉烦恼）
         │                 │                │
         │    ┌────────────┼────────────┐    │
         │    │            │            │    │
         ▼    ▼            ▼            ▼    ▼
     ACTION_CLICK    ACTION_DRAG     EMOTION_LIE
    （点击反应）     （拖拽跟随）    （躺平发呆）
```

### 2.2 状态转换矩阵

| 触发事件 | 当前状态 → 目标状态 | 转换条件 | 优先级 |
|---|---|---|---:|
| 页面加载完成 | 无 → IDLE_WALK | 初始化 | — |
| 用户点击乌萨奇 | *任何* → ACTION_CLICK | mousedown/touchend | P0（最高） |
| 用户拖拽乌萨奇 | *任何* → ACTION_DRAG | mousedown + mousemove > 10px | P0 |
| 用户松开（拖拽结束） | ACTION_DRAG → IDLE_WALK | mouseup/touchend | P0 |
| 点击"粉碎它！"按钮 | IDLE_* → ACTION_EAT | 烦碍文字飞到位置后 | P1 |
| 吃完动作完成 | ACTION_EAT → IDLE_WALK | 动画播放完毕 | 自动 |
| 点击反应完成 | ACTION_CLICK → IDLE_WALK | 动画播放完毕 | 自动 |
| 无操作超时（>15s） | IDLE_WALK → EMOTION_LIE | 定时器触发 | P2 |
| 用户再次交互（躺平时） | EMOTION_LIE → ACTION_CLICK | 任意点击 | P0 |
| 撞到屏幕边界 | IDLE_WALK → IDLE_WALK | 位置越界 | 内部处理 |

## 三、各状态的详细动画规格

### 3.1 IDLE_WALK — 巡逻漫游状态（默认常态）

**触发时机：** 初始进入、任何动作结束后回到此状态

**视觉表现：**
- 乌萨奇以固定高度在屏幕内自由移动
- 移动速度：1.5~2.5 px/frame（可在配置中调整）
- 方向：随机角度偏向水平（±72°），每 3~7 秒随机变向一次
- 碰撞屏幕边缘时反弹 + 小幅漂移扰动

**精灵图编排：**

| 子阶段 | 使用帧 | 播放节奏 | CSS/JS 动效叠加 |
|---|---|---|---|
| 向右移动 | `[1,2,3,4]` 循环 | 每 180ms 切一帧 | `usagi-walk-wobble` 抖动 + `facing-right` |
| 向左移动 | `[1,2,3,4]` 循环 | 每 180ms 切一帧 | `usagi-walk-wobble` 抖动 + `facing-left`（scaleX(-1)镜像） |
| 边界碰撞瞬间 | `[19]` 单帧闪现 | 持续 200ms | 轻微 `bounce` 缩放弹跳 |

**音效：**
- 无持续音效（静默漫游）
- 碰撞边界时可低音量播放 `sit.wav`（概率 30%）

**行为规则：**
- 走路期间保持 Z-index 在内容层之上但在 UI 层之下
- 夸夸气泡容器实时跟踪其当前位置
- 窗口 resize 时自动钳位到可视范围内

---

### 3.2 IDLE_STAND — 静止站立（可选降级状态）

**触发时机：** 当用户希望暂停走动时（如通过按钮或设置）

**视觉表现：**
- 固定于当前位置，不再位移
- 保持呼吸感浮动动画

**精灵图编排：**

| 子阶段 | 使用帧 | 播放节奏 | 叠加效果 |
|---|---|---|---|
| 平静呼吸 | `[1→2→3→4→5→6→7→8]` 循环 | 每 300ms 一帧 | `usagi-idle` 浮动（translateY ±8px） |
| 偶尔眨眼/抽动 | 插入 `[27]`（嘟嘴） | 随机间隔 8~15秒出现一次 | 持续 600ms 后恢复正常 |

**音效：** 无

---

### 3.3 ACTION_CLICK — 点击互动反应

**触发时机：** 用户单击/触摸乌萨奇本体

**视觉表现：**
- 立即停止走动 → 播放跳跃动画 → 落地后恢复走动
- 整个过程 0.6~0.8 秒

**精灵图编排（完整序列）：**

```
时间轴（总计 ~700ms）：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━▶

  0ms       100ms      250ms      400ms      550ms      700ms
  │         │          │          │          │          │
  ▼         ▼          ▼          ▼          ▼          ▼
[起跳预备] [上升中]   [最高点]    [下落中]    [落地]     [恢复]
  │         │          │          │          │          │
[4]       [19]↑      [6]★       [5]↓       [1]       [idle]
           (开心举   (最兴奋    (微笑回落  (正常站    (回到
           手帧)      张嘴帧)    帧)        立帧)      待机)
```

| 时间段 | 使用帧 | GSAP 运动参数 | CSS 类 |
|---|---|---|---|
| 0~100ms | `shime4` | `y: 0 → -15px, scale: 1 → 1.03` | 移除 walk，准备 jump |
| 100~250ms | `shime19` → `shime6` | `y: -15px → -40px, scale: 1.05~1.08` | `usagi-jump-anim` |
| 250~400ms | `shime6` → `shime5` | `y: -40px → -10px` | 持续 jump |
| 400~550ms | `shime5` → `shime1` | `y: -10px → 0px, scale: 1` | jump 结束过渡 |
| 550ms+ | idle 帧 | 弹性缓动回位 | 清除 action 类 |

**音效编排：**

| 时机 | 音效 | 音量 | 说明 |
|---|---|---:|---|
| 起跳瞬间（0ms） | `start.wav` | 0.6 | "呀哈"起始音 |
| 落地（500ms） | `sit.wav` | 0.4 | 轻轻落地的声音 |
| 连击加成（300ms内再点） | `double.wav` | 0.7 | 更高亢的反应音 |

**连击系统设计：**
```
点击次数 → 反应强度：
  1次：普通跳跃 + start.wav
  2次（<1s内）：更高跳跃(+-50px) + double.wav + 帧[8]插入
  3次+（<1s内）：超级跳跃(+-70px) + double.wav + 连续[6][7][8]快切 + 屏幕微震
  冷却时间：每次连击后 1.5s 重置计数器
```

### 3.4 ACTION_DRAG — 拖拽跟随

**触发时机：** mousedown 后鼠标/手指移动距离超过 10px

**视觉表现：**
- 乌萨奇跟随指针/触点移动
- 松开后根据抛出速度惯性滑行一段距离
- 如果拖拽到屏幕边缘则自动吸附在内侧

**精灵图编排：**

| 子阶段 | 使用帧 | 说明 |
|---|---|---|
| 拖拽中被抓取 | `[27]`（嘟嘴/不情愿脸）或 `[1]` | 表示"被抓了"的感觉 |
| 拖拽中移动 | `[9~12]` 循环 | 朝拖拽方向跑动的样子 |
| 高速拖拽 | `[17~18]`（带闪光尾巴） | 速度快时切换到"冲刺"帧 |
| 松开瞬间 | `[19]`（开心举手） | "飞出去了！"的感觉 |
| 惯性滑行中 | `[1~4]` 循环 | 自然减速 |
| 停止 | 回到 idle | — |

**物理参数：**

| 参数 | 推荐值 | 说明 |
|---|---|---|
| 拖拽灵敏度 | 1:1 跟随指针 | 无延迟跟随 |
| 最大拖拽速度 | 15 px/frame | 限制最大速度防止飞出 |
| 惯性系数 | 0.92 | 松开后速度每帧衰减 8% |
| 最小停止阈值 | 0.3 px/frame | 低于此值完全停止 |
| 边界弹性 | 0.5 | 撞墙后反弹保留 50% 速度反向 |

**音效：**

| 时机 | 音效 |
|---|---|
| 开始拖拽 | `start.wav`（短促版） |
| 高速甩动 | `double.wav`（每 300ms 最多播一次） |
| 松开 | `sit.wav` |
| 惯性撞墙 | `sit.wav`（音量减半） |

**交互细节：**
- 拖拽期间暂停走动引擎和待机动画
- 拖拽时 cursor 设为 `grabbing`
- 拖拽层 z-index 提升（确保在最上层）
- 移动端触摸支持：touchmove 阻止默认滚动

### 3.5 ACTION_EAT — 吃掉烦恼

**触发时机：** 烦恼文字碎裂粒子到达乌萨奇位置后

**这是整个产品核心仪式感动作，需要最有分量。**

**完整时序编排：**

```

═══════════════════════════════════════════════════════▶

 ACT 1 — 发现猎物（~400ms）
 ─────────────────────────────────
  0ms        150ms       350ms       400ms
  │          │           │           │
  ▼          ▼           ▼           ▼
[警觉]    [转身面对]   [准备扑]    [蓄力完]
  │          │           │           │
[4]→[5]   [9-14侧身]   [19]举手    [8]张嘴
           （朝向粒子    （兴奋      （大口
           来源方向）     准备）      等待）

 ACT 2 — 吞噬（~600ms）
 ─────────────────────────
  400ms      500ms       700ms      1000ms
  │          │           │           │
  ▼          ▼           ▼           ▼
[大口咬]   [咀嚼x3]    [吞咽]     [满足]
  │          │           │           │
[8]→[6]   [15-16]     [20]       [19]→[5]
 (张嘴咬)  (背身嚼)   (咽下去)   (开心满足)

 ACT 3 — 夸夸前摇（~1200ms，与气泡显示并行）
 ─────────────────────────────────────
  1000ms     1300ms      1800ms      2200ms
  │          │           │           │

  │          │           │           │
```

**音效精密编排：**

| 时间点 | 音效 | 音量 | 说明 |
|---|---|---:|---|
| 450ms（咬合瞬间） | `5.wav`（从头播放） | 0.8 | 咀嚼/吃的核心音效 |
| 750ms（吞咽） | `5.wav`（重置播放） | 0.5 | 第二次轻量确认 |
| 1100ms（满足叹气） | `sit.wav` | 0.4 | 吃完的满足声 |

**GSAP 运动叠加：**

| 时段 | 运动效果 | 参数 |
|---|---|---|
| Act 1 全程 | 身体微微前倾朝向粒子来源方向 | `rotation: -5°~5°` |
| 咬合瞬间（450ms） | 整体放大 | `scale: 1 → 1.15 → 1`（弹性） |
| 咀嚼期 | 快速缩放抖动 | `scale: 1±0.08` @ 120ms 循环 |
| 吞咽后 | 满足地晃动 | `rotate: ±3°` @ 400ms 衰减 |

### 3.6 EMOTION_LIE — 躺平发呆（超时触发）

**触发时机：** 连续 15~25 秒无用户交互

**设计意图：** 让乌萨奇有"生活感"，不是永远精力充沛的机器人。长时间不理它会自己找乐子——或者干脆躺平。这反而增加了用户想戳它的欲望。

**两种躺法随机选一：**

#### 方案 A — 仰面躺倒（"摆烂"模式）

| 子阶段 | 使用帧 | 持续 | 说明 |
|---|---|---|---|
| 躺倒过渡 | `[4] → [23]` | 500ms | 从站立慢慢向后倒 |
| 躺平呼吸 | `[23] ↔ [24]` | 每 2s 循环 | 仰面↔侧卧交替 |
| 偶尔抽动 | 插入 `[27]` | 随机 | 躺着也不老实，嘟嘴一下 |
| 用户点击唤醒 | `[24] → [19] → [1]` | 800ms | 受惊弹起 → 开心恢复 |

#### 方案 B — 侧卧小睡（"充电"模式）

| 子阶段 | 使用帧 | 持续 | 说明 |
|---|---|---|---|
| 坐下过渡 | `[1] → [20] → [24]` | 700ms | 慢慢坐下 → 趴下 |
| 睡眠呼吸 | `[24] → [25] → [26] → [25]` 循环 | 每 2.5s 一轮 | 闭眼↔半睁眼↔打盹 |
| ZZz 特效 | CSS 动画生成 | 持续 | 头顶飘出 "Z..." 文字 |
| 用户触碰唤醒 | `[26] → [4] → [19]` | 600ms | 揉眼 → 惊醒 → 开心 |

**音效：**

| 时机 | 音效 | 说明 |
|---|---|---|
| 躺倒/坐下 | `sit.wav` | 轻柔的"噗通"声 |
| 睡着后偶尔 | `start.wav`（音量 0.3） | 说梦话般的"呀哈" |
| 被唤醒 | `double.wav` | 惊醒的夸张反应 |

**睡眠状态下的特殊行为（增加趣味性）：**
- 眼皮缓慢开合（帧 25 ↔ 26 交替）
- 身体随呼吸微微起伏（CSS translateY ±3px）
- 偶尔腿蹬一下（随机 5~8 秒一次）
- 如果此时用户输入文字点了粉碎 → 乌萨奇会被声音"吵醒"，先揉眼再吃

## 四、复合场景编排（Cross-State Scenarios）

### 4.1 场景 A：用户一边输入文字，乌萨奇一边走到旁边偷看

```
用户聚焦输入框
    ↓
乌萨奇感知到输入活动（通过 input 事件监听）
    ↓
如果乌萨奇距离输入框 > 300px：
    → 以 2x 速度走向输入框附近（停在框外 80px 处）
    → 切换帧为 [9~14] 侧身，头朝向输入框方向
    → 到达后停顿 1 秒，表现出好奇（帧 [4] 歪头）
    ↓
用户继续输入中：
    → 乌萨奇保持静止观察，偶尔用 [27] 嘟嘴表示"在看但看不懂"
    → 每输入 10 个字符，身体微微前倾（CSS rotate 3°）
    ↓
用户点击"粉碎它！"
    → 进入 ACTION_EAT 流程（见 3.5 节）
```

### 4.2 场景 B：用户连续快速点击（连击狂欢）

```
第 1 次点击 → 普通 ACTION_CLICK（跳跃 + start.wav）
第 2 次点击（<1s内）→ 强化跳跃（高度+50%）+ double.wav + 帧[8]插入
第 3 次点击（<1s内）→ 超级跳跃（高度+100%）+ double.wav
                          + 屏幕 shake（CSS translate 随机 ±3px × 3次）
                          + 周围爆出 6~8 个小心形粒子（CSS 生成）
第 4 次点击（<1s内）→ 乌萨奇"受不了了"
                          → 切换 [27] 嘟嘴帧 1.5 秒
                          + 气泡显示："别戳啦！(>_<)"
                          + 进入 3 秒冷却期（不响应点击）

冷却结束后 → 恢复正常响应
```

### 4.3 场景 C：拖拽扔出去

```
mousedown（抓住）
    ↓
帧切换为 [27]（不情愿脸）/ [1]（呆滞）
    ↓ cursor: grabbing
mousemove（开始拖动）
    ↓
根据拖拽速度选择帧：
  低速（< 5px/frame）→ [1~4] 正常跟随
  中速（5~10px/frame）→ [9~12] 侧身跑
  高速（> 10px/frame）→ [17~18] 尾巴闪光冲刺
    ↓
mouseup（释放）
    ↓
帧闪 [19]（开心举手）— "我飞啦！"
    ↓
按释放瞬间的速度计算惯性向量
    ↓
惯性滑行（摩擦系数 0.92/帧），同时帧 [1~4] 循环
    ↓
速度衰减到 < 0.3 或碰到墙壁
    ↓
播放 sit.wav（落地声）
    ↓
短暂停顿 500ms（帧 [20] 站立喘气）
    ↓
恢复 IDLE_WALK
```

### 4.4 场景 D：夸夸气泡显示时的协同表演

```
ACTION_EAT 结束 → praiseBubble.show() 被调用
    ↓
气泡从乌萨头顶上方弹出（GSAP elastic.out）
    ↓
与此同时乌萨奇的表演：
    ① 气泡出现瞬间（0ms）→ 帧 [19] 开心举手
    ② 气泡打字中（0~2000ms）→ 帧 [1~6] 慢循环，身体轻轻摇摆
    ③ 打字完成（2000ms+）→ 帧 [5] 满足微笑，定格 2 秒
    ④ 气泡淡出后 → 恢复 IDLE_WALK
    ↓
能量波（EnergyWave）在气泡 show callback 时同步发射
    ↓
颜色根据 praiseType 映射：
  warmth  → 金橙色 (#FFB64C)
  gentle  → 樱花粉 (#FFC8DC)
  funny   → 天蓝 (#B4E6FF)
  yaha    → 明黄 (#FFD700)
```

### 4.5 场景 E：窗口失焦 / 用户切换 Tab

```
document.visibilitychange === 'hidden'
    ↓
降低动画质量以节省资源：
  1. 粒子背景降低到 14 个（原 28 个的一半）
  2. 乌萨奇走动速度降到 50%
  3. 帧率从 60fps 降至 30fps（requestAnimationFrame 跳帧）
  4. 停止播放所有音效
    ↓
document.visibilitychange === 'visible'
    ↓
逐步恢复（500ms 内平滑过渡回全画质）
```

### 4.6 场景 F：移动端触摸适配

| 桌面事件 | 移动端等效 | 特殊处理 |
|---|---|---|
| click | touchend | 阻止 300ms 延迟（touch-action: manipulation） |
| drag (mouse) | touchmove | 阻止页面滚动（e.preventDefault()） |
| hover 状态 | 无等效 | 移除所有 :hover 依赖样式 |
| 双击/连击 | 快速双 tap | 检测两次 touchend 间隔 < 300ms |
| 右键菜单 | 长按 > 500ms | 触发"抱起"效果（乌萨奇被抱在中间放大）|
