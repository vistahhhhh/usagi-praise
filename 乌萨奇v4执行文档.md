### 1.2 Rail 数据结构（JS 实现）

```javascript
// usagi-character.js 中定义
var RAILS = [
  { id: 0, name: 'ceiling',   yMinPct: 0,  yMaxPct: 0.12, anchorY: 'top' },
  { id: 1, name: 'high',      yMinPct: 0.12, yMaxPct: 0.35, anchorY: null }, // 空中区无锚点
  { id: 2, name: 'mid',       yMinPct: 0.35, yMaxPct: 0.55, anchorY: null },
  { id: 3, name: 'low',       yMinPct: 0.55, yMaxPct: 0.80, anchorY: null },
  { id: 4, name: 'floor',     yMinPct: 0.80, yMaxPct: 1.0,  anchorY: 'bottom' }
];

// 获取某条 Rail 的中心 Y 坐标（像素值）
function getRailCenterY(railId) {
  var rail = RAILS[railId];
  var h = window.innerHeight;
  return h * (rail.yMinPct + rail.yMaxPct) / 2;
}

// 判断一个 Y 像素值属于哪条 Rail
function getRailFromY(yPx) {
  var h = window.innerHeight;
  var pct = yPx / h;
  for (var i = 0; i < RAILS.length; i++) {
    if (pct >= RAILS[i].yMinPct && pct < RAILS[i].yMaxPct) return i;
  }
  return 4; // 默认地板
}
```

### 1.3 Rail 间转换规则（改编自原始攀爬/掉落）

```
向上走（只能通过墙壁攀爬）:
  当前 Rail → 跑到左/右墙边 → ClimbWall([12][13][14])向上爬 → 到达目标Rail Y值 → 弹出离开墙 → 到达新 Rail

向下走（两种方式）:
  方式A - 从墙松手: 在墙上时选择 FallFromWall → 重力掉落到下方 Rail → Bouncing([18][19]) → Stand([1])
  方式B - 受惊跳轨: 在Rail内随机触发(约2%/秒概率) → [4]翻白眼 → 自由落体 → [11]坐着落地 → [1]站起

水平移动（Rail内部）:
  与原始完全一致: Walk(-2)/Run(-4)/Dash(-8) 纯水平 + [1][2][3] 帧
```

## 五、行为选择器算法（从原始 Frequency Weighted Random 直接翻译）

### 5.1 核心算法

```javascript
/**
 * 从可用行为列表中按频度加权随机选一个
 * 直接翻译自原始 Shimeji Java 引擎的选择逻辑
 *
 * @param {Array} candidates - [{name, frequency}, ...]
 * @param {Object} conditions - 额外的条件过滤（可选）
 * @returns {string} 选中的 behavior 名称
 */
function selectBehavior(candidates, conditions) {
  // 1. 过滤: 只保留满足条件的候选者
  var valid = candidates.filter(function(b) {
    if (conditions && b.condition && !b.condition(conditions)) return false;
    if (b.hidden) return false;  // Hidden 行为不参与直接选择
    return true;
  });

  if (valid.length === 0) return null;

  // 2. 计算总权重
  var totalWeight = 0;
  for (var i = 0; i < valid.length; i++) {
    totalWeight += valid[i].frequency;
  }

  // 3. 加权随机掷骰
  var roll = Math.random() * totalWeight;
  var cumulative = 0;
  for (var j = 0; j < valid.length; j++) {
    cumulative += valid[j].frequency;
    if (roll < cumulative) {
      return valid[j].name;
    }
  }

  return valid[valid.length - 1].name;  // 兜底
}
```

### 5.3 行为执行生命周期

```
每个 Behavior 的完整生命周期:

  1. SELECT   → selectBehavior() 从池子中加权随机选一个
  2. ENTER    → 执行入口动作（如 WalkAndGrabLeftWall 先 Run 到左边）
  3. RUN      → 执行主体动作序列（按 Action 定义的一步步 Pose 执行）
  4. COMPLETE → 主体动作完成
  5. NEXT     → 查看 NextBehavior 链（如果有），selectBehavior() 选下一个
               → 如果没有 NextBehavior 链: 回到当前 Rail 的主行为池重新 SELECT

特殊情况 - 中断:
  如果在 RUN 阶段收到外部事件(点击/focus/提交):
    → 暂存当前行为状态（currentBehavior, progress, rail）
    → 执行交互行为(ACTION_HA / ACTION_EAT / PEEK_INPUT)
    → 交互结束后恢复暂存的状态（或重新 SELECT 如果状态已失效）
```
