### 原始 Shimeji 的核心架构（我们必须借鉴的）

#### 1. "表面附着"状态机 —— 一切行为的根基

```
Shimeji 不用"状态"来决定行为，而是用"当前附着在哪个物理表面上"来决定：

屏幕 = 一个有边界的物理盒子：

   +-- 天花板 (ceiling) --------------------------------+
   |  行为: 抓住天花板 / 在天花板上爬行 / 掉落         |
   +----------------------------------------------------+
                                                    |
   |  左墙壁 <-----------> 右墙壁                        |
   |  行为: 抓墙 / 攀爬(上/下) / 从墙掉落               |
                                                    |
   +----------------------------------------------------+
   |  地板 (floor) -------------------------------------+
   |  行为: 站立发呆 / 坐下发呆 / 趴下                   |
   |       走 / 跑 / 爬行(蹭) / 攀爬墙壁                 |
   |       走到角落坐下 / 跳向墙壁                       |
   +----------------------------------------------------+
```

**关键发现: 走和跑都是纯水平的!**

看 Actions.xml 定义:
- 歩く(walk): 移動速度="-2,0" --> 纯水平!
- 走る(run):  移動速度="-4,0" --> 纯水平!
- 猛ダッシュ(sprint): 移動速度="-8,0" --> 还是纯水平!

垂直方向的运动只通过以下方式发生: 攀爬墙壁 / 跳跃 / 掉落(重力)

#### 2. 频率权重随机选择器 (Frequency Weighted Random)

每个"表面"下列出所有可用行为，每个行为有一个 頻度(frequency weight):
- 立ってボーっとする 頻度="200" (站立发呆)
- 座ってボーっとする 頻度="200" (坐下発呆)
- ワークエリアの下辺を歩く 頻度="100" (在地板上走)
- ワークエリアの下辺を走る 頻度="100" (在地板上跑)
- ワークエリアの下辺でずりずり 頻度="10" (在地板上爬行/蹭)

系统根据权重比例随机选一个执行。总权重=610, 站立概率=200/610约33%, 爬行只有10/610约1.6%

#### 3. 动作链式组合 (NextBehavior Chaining)

一个行为完成后，可以链接到下一个行为:
```xml
<Behavior Name="座ってボーっとする" Frequency="200">
  <NextBehavior Add="true">
    <BehaviorReference Name="SitWhileDanglingLegs" Frequency="100" />
    <BehaviorReference Name="LieDown" Frequency="100" />
  </NextBehavior>
</Behavior>
```
坐下发呆结束后, 50%概率晃腿, 50%概率趴下。

#### 4. 墙壁交互的两套独立体系 (关键!)

从代码中明确看到: 撞墙 和 爬墙 是完全独立的两个行为分支!

**体系 A - 撞墙反应(瞬间的):**
通过 JumpFromLeftWall / JumpFromRightWall 实现
跳跃起跳 → 落地后 壁に掴まる(抓墙单帧) → 可能接 壁から落ちる(掉落)

**体系 B - 墙壁攀爬(持续的):**
```xml
<Behavior Name="ワークエリアの壁を登る"> <!-- 走到墙边 → 攀爬UP -->
<Behavior Name="ワークエリアの壁を途中まで登る"> <!-- 攀爬一半停下 -->
```
使用帧 [12][13][14] 循环向上/向下爬

#### 5. 完整的"楼层"概念

Shimeji 的屏幕天然就是多楼层的:
- Floor(地板) = 屏幕底部, 主要活动区
- Wall(侧墙) = 可以攀爬上下以改变高度
- Ceiling(天花板) = 屏幕顶部, 可以趴在上面爬

角色在这些表面之间转换的方式:
```
地板 --[走到墙边+攀爬]--> 侧墙上
侧墙上 --[继续向上爬]--> 天花板
天花板 --[沿天花板爬行]--> 天花板各位置
天花板 --[松手掉落/跳下]--> 地板
地板 --[跳跃]--> 侧墙中部
侧墙 --[松手掉落]--> 地板
```

## 二、运动系统 v3 - 横向跑 + 格子楼层 + 墙壁攀爬

### 核心转变: 从"自由移动"到"表面附着"

```
X v2 错误理解: 乌萨奇在整个屏幕内 free-roam (任意方向匀速+边界反弹)
O v3 正确方案: 乌萨奇附着在"虚拟表面"上, 只能沿表面或跳跃
```

屏幕被划分为 5 条水平"轨道"(Floor/Rails):

```
     Rail 0: ======================  (天花板区 y: 0~15%)
              可: 贴天花板[23-25] / 沿天花板爬
                |
     Rail 1: ------------------------  (高区 y: 15%~40%)
              可: 爬墙经过 / 跳跃中转 / 偶尔贴墙休息
                |
     Rail 2: ------------------------  (中区 y: 40%~60%)  
              可: 主要活动区之一 / 偷看输入框最佳高度
                |
     Rail 3: ------------------------  (低区 y: 60%~80%)
              可: 主活动区 / 摔倒后坐在这里
                |
     Rail 4: ======================  (地板区 y: 80%~100%)
              可: 主要奔跑/行走 / 坐下发呆 / 趴下 / 攀爬起点
```

### 2.1 横向运动 (Rail 内部)

乌萨奇在同一 Rail 内只做水平移动 (完全还原原始 Shimeji):

| 运动模式 | 速度(px/frame) | 使用帧 | 触发权重 |
|---|---:|---|---:|
| **站定** | 0 | [1] 或 [5] | 200 (默认态) |
| **慢走** | 0.8~1.2 | [2]<->[3] @200ms | 100 |
| **奔跑** | 2.0~3.0 | [2]<->[3] @130ms | 100 (主要模式) |
| **冲刺** | 4.0~5.0 | [2]<->[3] @90ms | 20 |

方向切换: 通过 scaleX(-1) 镜像翻转实现向右跑。不是斜向跑!

### 2.2 Rail 间跳跃 (纵向切换)

#### 向下跳 (受惊/滑倒)

```
触发: 随机(每秒约2%概率) 或边缘特殊行为

序列: [当前奔跑] --> [4](失控翻白眼!) --> 自由落体到下一Rail 
    --> 落地[11](坐着震惊) --> [1](站起来) --> 恢复奔跑

音效: 起跳无声 / 落地 sit.wav vol 0.5

视觉: 起跳时 rotate(-15deg) / 空中可能翻滚 / 落地 bounce缓冲(elastic.out)
```

#### 向上跳 - 只能通过墙壁!

```
乌萨奇不能直接向上跳。要往上走必须:
  1. 横向跑到屏幕左侧或右侧边缘
  2. 选择"攀爬"行为(而不是"撞墙")
  3. 用[12][13][14]帧沿墙壁向上爬
  4. 到达目标Rail高度后
  5. 选择"离开墙壁" --> 以小跳距离弹出到该Rail

这就是原始 Shimeji 的 よじのぼる (攀爬) 系统!
```

### 2.3 墙壁交互双轨制

用户强调的关键修正: 撞墙 != 贴墙爬, 两套独立体系!

#### 轨A: 撞墙反应 (Wall Bump) - 快速、瞬间

```
触发: 横向跑碰到屏幕边缘

两种撞法随机(各50%):

  撞法1 - 翻白眼撞:
  ├── 撞击(0ms): [9]左墙 or [10]右墙 (翻白眼! 身体后仰)
  ├── 音效: sit.wav vol 0.6 ("咚!")
  ├── 后果(0~400ms): GSAP 弹回20px + back.out(2)缓动
  └── 恢复: 掉头反向跑 [2][3]
      整个过程不超过500ms

  撞法2 - 轻微碰壁:
  ├── 碰壁(0ms): [19]或[22] (轻微倾斜, 无翻白眼或程度轻)
  ├── 音效: start.wav vol 0.3 (短促"嘟")
  └── 恢复: 直接掉头反向跑
```

#### 轨B: 墙壁附着与攀爬 (Wall Cling & Climb) - 持续、主动

```
触发: 乌萨奇主动选择走到墙边(不是被迫撞上的)
      在边缘行为选择中选中了"攀爬"选项

阶段1 - 附着(抓墙):
  帧: [12](粘在墙上闭嘴), 持续300~800ms
  表现: 身体贴在墙边缘, 微微颤动 +/-2px
  音效: start.wav vol 0.2

阶段2 - 攀爬(向上或向下):
  帧: [12] --> [13](张嘴) --> [14](闭嘴) 循环
  节奏: 每250ms一帧(费力感)
  移动: Y方向 +/-1.5 px/frame
  音效: 攀爬中静默(偶尔8%概率 start.wav vol 0.15)

阶段3 - 离开墙:
  到达目标高度后
  帧: [4](蹬墙弹出!) 或 [1](轻轻松开)
  弹出15~30px 到新的Rail
  音效: start.wav vol 0.3

特殊情况 - 在墙上发呆:
  如果攀爬中途停下来:
  [12]<->[13]<->[14] 慢循环(每500ms一帧), 持续2~5秒
  之后继续爬 或 掉下来

变体 - 从墙掉落:
  在墙上附着超过一定时间 OR 用户点击它
  [12/13/14] --> 松手 --> 重力掉落(GSAP power2.in加速)
  --> 落到Rail 4(地板) --> [19]滑倒/[11]坐下/[18]摔僵(随机)
  --> [1]站起 --> 恢复奔跑
  音效: 掉落开始 start.wav + 落地 sit.wav
```

### 3.3 交互行为 (优先级最高, 可中断任何上述行为)

#### 点击 -> "HA!" 反应

```
无论乌萨奇在做什么(跑/爬墙/贴天花板/躺着...), 点击即触发:

标准HA! (~1500ms总时长):

  任何当前帧 --> 立刻中断 --> 
  [26]抱胸闭嘴预备(0~300ms) -->
  [27]抱胸张嘴"呀哈!!!" ★ (播放5.wav, GSAP scale 1->1.2弹性膨胀, 震动+/-4px) -->
  [28]抱胸闭嘴收尾(音频结束后~300ms, scale回正+余震衰减) -->
  回到被中断前的行为/状态

特殊情况 - 在不同位置被戳:
  🏃 跑途中被戳 -> HA!完毕后继续跑(不停下)
  🧗 爬墙时被戳 -> 先从墙掉落 -> 空中/落地做HA!
  📌 贴天花板被戳 -> 掉落+空中HA!+落地
  😴 躺着/坐着被戳 -> 坐着/躺着的姿态做HA!(更搞笑因为姿势不对)

连击增强:
  第2次(<1.2s): double.wav + scale1.3 + 屏幕震动
  第3次(<1.2s): HA! + 接扭屁股[15-17]循环1.5s + 星星粒子
  第4次+: [30]生气蹲 + "别戳啦!" + 3s冷却
```

#### 吃烦恼 (ACTION_EAT)

```
粉碎按钮点击后触发:
  1. 烦碍文字飞向乌萨奇当前位置(无论在哪条Rail)
  2. 乌萨奇感知到 -> 中断当前行为
  3. 如果不在地板(Rail 4): 先掉落/跳到Rail 4 (吃东西要在地上吃!)
  4. Eat序列: [5]警觉 -> [26]蓄力 -> [27]咬!(5.wav) 
     -> [15][16][17]扭屁股嚼(double.wav) -> [28]收尾(sit.wav) -> [5]满足
  5. 夸夸气泡弹出(乌萨奇旁边扭屁股或满足笑)
```

#### 偷看输入框 (PEEK_INPUT)

```
用户focus输入框 + 乌萨奇距离>200px时:
  1. 如果在Rail 0/1(太高): 先掉落到Rail 2或3
  2. 计算不遮挡停靠位置(输入框左侧或右侧,留160px间隙)
  3. 横向跑过去([2][3]奔跑,速度2.5x)
  4. 到达后:
     初始0~5s: [5]低头偷看(偶尔插[1]站直)
     5~15s: [7]或[8]悬空坐下等(无聊了)
     15s+: 偶尔[15-17]扭屁股打发时间(8%/s概率)
     检测即将提交: [26]HA!预备(身体紧绷)
  5. z-index始终保持9(低于UI层10), 绝不遮挡输入框!
```

## 五、实施路线图 v3

### Phase 0: 基础设施 (必须先做)
- [ ] **Rail楼层系统**: 屏幕Y轴划分为5条Rail(0~4), 每条有Y范围定义
- [ ] **表面附着检测**: 乌萨奇知道自己所在Rail、是否接触墙壁/天花板
- [ ] **SPRITE_CONFIG v3**: 按帧分组重新定义所有帧常量

### Phase 1: 核心运动
- [ ] **横向奔跑引擎重写**: 纯水平移动+[2][3]帧切换+方向镜像
